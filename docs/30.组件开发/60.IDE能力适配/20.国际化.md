# 国际化

## 1. 功能说明

依赖库组件支持在IDE引入后，通过[国际化多语言](../../../../../20.应用开发/60.可视化开发界面/70.更多设置/50.国际化多语言.md)功能配置组件的系统文案。

## 2. 功能实现

### 2.1 组件内部获取国际语言

在组件内部，可以通过以下方式获取相关信息：

- 获取当前语言环境

    ```
    window.$global.i18nInfo.locale
    ```

- 获取所有的语言包

    ```
    window.$global.i18nInfo.messages
    ```

<img src="https://community.codewave.163.com/libraryUpload/app/CH1FdglXH_Eswibt5AYCbFg0OFmpDoKuEmP6_xeNXHE9Tclp_eTfBasXc58q7gp5vC-Q_E3uQ0WjA8S8jHfhUw_20241122153743204_ori.png" class="imgStyle" style="" />

### 2.2 组件文案上传到 IDE 国际化配置

1.  依赖库组件目录下增加 `i18n` 目录。

    ```
    |-- src
    |---- components
    |------- cwd-capluse
    |---------- i18n
    |------------- zh-CN.json 
    |------------- en-US.json
    ```

    <img src="https://community.codewave.163.com/libraryUpload/app/CH1FdglXH_Eswibt5AYCbIyKvoPS6OyKiO3N3J7r5oQ9Tclp_eTfBasXc58q7gp5vC-Q_E3uQ0WjA8S8jHfhUw_20241122153742784_ori.png" class="imgStyle" style="" />

1.  配置文案。本示例将配置文案写在zh-CN.json 文件中，并在 index.vue 处引用。

    - 配置文案示例如下：

    ```json
    //仅支持二级结构，key 建议带上组件的前缀。
    {
    "cwd-capluse.text": "这个是胶囊选择器的国际化测试文案"
    }
    ```

    <img src="https://community.codewave.163.com/libraryUpload/app/CH1FdglXH_Eswibt5AYCbON7LpL-r7PDz-nlGM-0SX89Tclp_eTfBasXc58q7gp5vC-Q_E3uQ0WjA8S8jHfhUw_20241122153740791_ori.png" class="imgStyle" style="" />

    - 在index.vue引入配置好的文案。

    <img src="https://community.codewave.163.com/libraryUpload/app/CH1FdglXH_Eswibt5AYCbJWpnCSoWkyqw7_st-FNcbs9Tclp_eTfBasXc58q7gp5vC-Q_E3uQ0WjA8S8jHfhUw_20241122153747595_ori.png" class="imgStyle" style="" />

2.  更改 vite.config.js 构建配置，增加 i18n 配置项：

    ```json
    // lcapPlugin 打包配置更改
    lcapPlugin({
        i18n: true,
    }),
    ```

    <img src="https://community.codewave.163.com/libraryUpload/app/CH1FdglXH_Eswibt5AYCbMjfjreOptSHtxW5u3QaDRQ9Tclp_eTfBasXc58q7gp5vC-Q_E3uQ0WjA8S8jHfhUw_20241122153801150_ori.png" class="imgStyle" style="" />

### 2.3 IDE内配置文案

以下步骤为依赖库打包发布并在IDE成功引入后的操作。

1.  点击 IDE 页面上方的“更多”按钮，在弹出的选项中，选择“国际化多语言”。

    <img src="https://community.codewave.163.com/libraryUpload/app/CH1FdglXH_Eswibt5AYCbDDDW0kVXR_3EZR1gtvcXbM9Tclp_eTfBasXc58q7gp5vC-Q_E3uQ0WjA8S8jHfhUw_20241122153801960_ori.png" class="imgStyle" style="" />

2. 配置依赖库组件内置文案。更多国际化多语言的配置说明及详细功能介绍，可参见：[国际化多语言](../../../../../20.应用开发/60.可视化开发界面/70.更多设置/50.国际化多语言.md)。

    <img src="https://community.codewave.163.com/libraryUpload/app/CH1FdglXH_Eswibt5AYCbA1a-SH82YJETGx67t3X9WE9Tclp_eTfBasXc58q7gp5vC-Q_E3uQ0WjA8S8jHfhUw_20241122153747707_ori.png" class="imgStyle" style="" />

> 注意：
> 依赖库组件内置文案是在依赖库组件开发时就已定义好的，无法在 IDE 内进行更改。例如上图所示，选中行中文文案为开发时定义好的无法在ID内更改；
> 但对于那些未定义的文案，可以自行配置。例如上图所示，中文文案对应的英文文案可以在 IDE 内设置。
